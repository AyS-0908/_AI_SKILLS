<skills_design_best_practices>
Best-practices summary — creating a Skill (apply going forward)

Structure
- One entry (SKILL.md) with YAML frontmatter (name, description only). Body = router + shared rules + ≤1 page.
- MECE across files: every rule in exactly one place. Mode files = mode-specific steps + one output template.
- Flat layout, kebab-case filenames, no spaces, no dev artifacts in the runtime folder.

Description (the single most important field — what makes the skill fire correctly)
- Three parts mandatory: WHAT + Trigger for: + Do NOT trigger for:.
- Name the adjacent skills/topics you explicitly exclude. Vague exclusions don't prevent co-fire.

Content discipline
- Lists and IF/THEN over prose. Tables for routing/priority.
- "Does deleting this sentence change behavior?" If no, delete.
- Don't restate what the LLM already knows or what MCP/docs inject every session.
- One output template per mode. Drop sections that add no value rather than padding.

Gotchas + error handling
- "## Gotchas" is the first body section of SKILL.md (or explicit "No known gotchas").
- Every external dependency (MCP, file read, API, web fetch) needs an explicit failure clause.
- Always treat HTTP 200 with success:false body as failure — never trust status code alone.

Sources & honesty
- Tag every knowledge source inline ([n8n MCP], [n8n doc], [internal memory]).
- Forbid claiming verifications that didn't happen.
- Distinguish durable (business behavior, contracts) from volatile (node names, params) — volatile is MCP/doc-resolved at runtime, never memorized.

Hygiene
- No secrets in or out of the skill.
- No over-engineering: every step needs both a business and technical "why" — otherwise cut.
- Separation of concerns enforced in the router (Specify ≠ Build ≠ Audit; never blend deliverables).
- Use modern native nodes/tools by default; flag custom code as the exception.

Validation before shipping
- Static: frontmatter present, links resolve, no filename typos.
- MECE grep: each shared rule appears once.
- Negative-trigger test: write 4+ near-miss prompts; confirm none co-fire.
- Live runs across all modes; cross-mode sanity (Specify never emits JSON, etc.).
