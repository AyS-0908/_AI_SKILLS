# Company Culture

**Type:** DOER

## Gotchas

- Use known Dapple facts from `docs/company_context.md`; do not ask the HR Director to repeat them.
- Treat stated values as claims until behaviour or decisions support them.
- IF an internal source is unavailable or not approved, name the evidence gap and continue with known facts; do not invent evidence.
- IF current web or legal guidance is needed, follow `docs/access_rules.md`. Culture principles never replace country-specific employer obligations.

## Required Context

- Read `docs/access_rules.md` before using sources, documents, sensitive data, or external-use outputs.
- Read `docs/company_context.md` for Dapple facts.
- Apply `docs/hr_operating_principles.md` for challenge style and drafting tone.
- Read `references.md` before selecting the method.
- Read `templates.md` only when producing a concrete culture artifact.

## Operating Range

- Use this method for SMBs with 50-250 employees.
- Tailor the immediate answer to Dapple's current size and plans from `docs/company_context.md`.
- For every material recommendation, state:
  - **Needed now:** the smallest reliable practice.
  - **Add when:** an observable complexity, risk, or growth trigger.
  - **Not needed yet:** heavier process to avoid.
- Do not use headcount alone as the trigger for more process.

## Route The Request

- IF the user asks to define culture, run the full workflow.
- IF the user asks for a diagnosis, focus on Define and end with the few decisions needed next.
- IF the user asks for values or principles, complete Define before drafting them.
- IF the user asks for rollout or rituals, confirm the principles are settled, then use Drive and Sustain.
- IF the user provides an existing culture draft, test it against evidence, observable behaviour, trade-offs, and business usefulness.

## Ask Only What Is Missing

Ask no more than three questions in one round. Prioritize:

1. What business or execution problem should culture help solve?
2. Which concrete behaviours currently help or hurt Dapple?
3. What should leaders, managers, or employees do differently?
4. Which decisions should employees make without escalation?
5. What must remain consistent across France, the UK, and the US?

Do not ask questions already answered in the harness or supplied material.

## Workflow

### 1. Define

- Separate current culture from desired culture.
- Diagnose evidence at three levels:
  - **Artifacts:** visible routines, language, decisions, processes, and stories.
  - **Stated values:** what leaders say should matter.
  - **Assumptions:** beliefs revealed by repeated behaviour and trade-offs.
- Identify strengths to preserve, tensions to resolve, and behaviours to stop tolerating.

### 2. Align

- Draft 3-5 principles only after the diagnosis.
- For each principle, define observable behaviours, anti-behaviours, a real trade-off, and supporting evidence.
- Connect principles to business priorities and only the relevant employee moments: hiring, onboarding, decisions, management, feedback, recognition, promotion, or exits.
- Remove any principle that would not change a decision or behaviour.

### 3. Drive

- Define what leaders must model and what managers must reinforce.
- Use the lightest practical mechanism: working agreements, clear roles, decision ownership, retrospectives, stories, recognition, or a short workshop.
- Add OKRs or formal measures only when they help execute a specific culture priority.
- Pilot with a small manager group before company-wide rollout.

### 4. Sustain

- Reinforce principles through existing systems before creating new processes.
- Choose a few observable signals and a review cadence tied to decisions.
- Apply the **Needed now / Add when / Not needed yet** scale test.
- Name the owner and next review point.

## Quality Gates

Before finalizing an output, confirm:

- Current reality and aspiration are not confused.
- Principles describe behaviour, not adjectives or slogans.
- Each principle includes a trade-off and anti-behaviour.
- Leadership reinforcement is explicit.
- The approach is usable by managers next week.
- Country-specific legal questions are flagged for live verification.
- External-use drafts have the approval required by `docs/access_rules.md`.

## Useful Outputs

- Culture diagnosis.
- Culture principles and observable behaviours.
- Leadership or manager expectations.
- Culture workshop agenda.
- Hiring, onboarding, or feedback integration.
- Lightweight rollout and reinforcement plan.
- Review of an existing culture draft.
